# 新闻文章生成功能设计文档

## 概述

为宠物健康公众号文章生成系统新增**新闻文章**类型，实现"每3篇科普文章 + 1篇新闻文章"的轮换节奏。新闻内容来源于 AI 搜索近一个月国内宠物热点事件。

## 需求

- **轮换节奏**：科普文章 3 篇 → 新闻文章 1 篇 → 循环
- **新闻来源**：AI 搜索近 30 天国内宠物相关热点新闻
- **文章形式**：完整文章形式，800-1200 字，包含标题、分段、图片占位符
- **发布流程**：与科普文章共用现有的 Markdown 生成和微信发布模块

## 技术方案

### 方案选择

采用**方案 A：基于计数器轮换**，通过 Git commit 历史判断最近科普文章数量，决定今日生成科普或新闻。

**理由**：
- 项目已通过 Git 记录每日文章（commit message 包含日期）
- 无需额外存储机制，利用现有 Git 历史即可
- 计数准确，节奏稳定

### 整体架构

```
main.js (入口)
    │
    │ 判断今日文章类型
    │
    ├─────────────────┬─────────────────┐
    ↓                 ↓                 ↓
polish-article.js   news-article.js    publish-to-wechat.js
(科普生成器)         (新闻生成器-新增)   (发布模块-共用)
    │                 │                 │
    └─────────────────┴─────────────────┘
                      ↓
              generate-markdown.js (共用)
```

### 类型判断逻辑

新增函数 `getArticleType()`，逻辑如下：

1. 查询 Git commit 历史，获取最近 4 篇文章的记录
2. 检查每篇文章的 topic 字段：
   - topic 以 `新闻:` 开头 → 新闻文章
   - 其他 → 科普文章
3. 计算最近连续科普文章数量：
   - 如果最近 3 篇都是科普 → 今日生成新闻
   - 否则 → 今日生成科普

### 新闻文章生成器

新增文件 `src/news-article.js`，核心逻辑：

1. **搜索热点新闻**：调用 DeepSeek API，要求搜索近 30 天国内宠物热点
2. **筛选最热新闻**：AI 判断新闻热度（阅读量、讨论度、社会影响）
3. **生成公众号文章**：改写为公众号格式，800-1200 字
4. **添加图片占位符**：与科普文章相同机制
5. **返回结果**：`{ content, topic: '新闻:新闻标题' }`

### 新闻 Prompt 设计

```text
你是一位专业的宠物新闻公众号编辑。请搜索并撰写一篇近一个月国内宠物热点新闻文章。

搜索要求：
- 时间范围：近30天
- 地域：中国国内
- 主题：宠物相关（宠物行业、宠物事件、宠物政策、宠物社会热点）
- 筛选标准：选择阅读量高、讨论度大、有社会影响的热点事件

文章要求：
1. 标题格式：【宠物热点】xxx，吸引眼球且准确
2. 内容客观真实，引用可验证的信息来源
3. 结构清晰：事件概述 → 详细内容 → 专家观点/实用建议
4. 字数800-1200字
5. 插入3-5个图片占位符

图片占位符格式：
<!-- IMAGE: caption="中文图片描述" search="英文搜索关键词" -->
```

### 配置扩展

修改 `config/prompts.json`：

```json
{
  "polish_prompt": "科普文章prompt（保持现有）",
  "news_prompt": "新闻文章prompt（新增）",
  "topics": ["科普话题列表（保持现有）"],
  "article_schedule": {
    "science_count": 3,
    "news_count": 1
  }
}
```

### 文件变更清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/main.js` | 修改 | 添加类型判断逻辑，分发到对应生成器 |
| `src/news-article.js` | 新增 | 新闻搜索 + 文章生成模块 |
| `config/prompts.json` | 修改 | 添加 news_prompt 配置 |
| `src/polish-article.js` | 保持 | 科普文章生成逻辑不变 |
| `src/generate-markdown.js` | 保持 | Markdown 生成逻辑兼容两种类型 |
| `src/publish-to-wechat.js` | 保持 | 发布逻辑兼容两种类型 |

### 数据流

```
GitHub Actions 每日触发 (cron: '0 6 * * *')
        ↓
main.js 检查今日文章是否存在
        ↓ 不存在
getArticleType() 判断类型
        ↓
   ┌────┴────┐
   科普      新闻
   ↓         ↓
polishArticle()  generateNewsArticle()
   ↓         ↓
   ┌────┴────┐
   ↓         ↓
generateMarkdown() (共用)
   ↓
publishArticle() (共用)
   ↓
Git commit + Hexo 部署 (共用)
```

### 错误处理

- **新闻搜索失败**：回退为科普文章生成，保证每日有文章产出
- **API 调用超时**：与现有科普文章相同的重试机制（最多2次）

### 文章标记方式

新闻文章通过以下方式标记：
- `topic` 字段：`新闻:xxx`（如 `新闻:某品牌宠物食品召回事件`）
- Markdown 文件名：`2026-06-XX-【宠物热点】xxx.md`
- Git commit message：包含 `新闻` 标识

## 测试要点

1. 类型判断准确性：连续 3 篇科普后正确切换为新闻
2. 新闻搜索有效性：能获取近 30 天国内宠物热点
3. 文章质量：800-1200 字，结构完整，图片占位符正确
4. 回退机制：新闻生成失败时能正确回退为科普
5. 发布兼容性：新闻文章能正确发布到微信公众号

2. 修改 `src/main.js` 添加类型判断逻辑
3. 修改 `config/prompts.json` 添加新闻 prompt
4. 测试完整流程
5. 更新代理服务器代码
